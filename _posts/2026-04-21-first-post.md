As an avid enthusiast of self-marketing and understanding the power of a good presentation, building a portfolio website has long been at the top of my list of 'things to do.' - collecting dust somewhere right between trying out that new recipe for pad thai and reading Atomic Habits. I had imagined that the endeavour would require long hours hunched over a laptop screen, debugging Node.js environments, correcting cryptic files, and suffering for my work. What I did not imagine was that I would have a working, presentable website by the end of my lunch. 

As a student of computer science, I am up to date with all the latest discoveries and revelations in the race for AI but what I did not expect was how easy it would be these days to create websites leading me to this conclusion - "The barrier between shipping a personal website in 2026 is not a tehcnical skill it is simply the willingness to begin."

This story begins, as I am sure many stories from now on will, with a conversation with AI. Claude suggested that I use GitHub Landing Pages, while ChatGPT expounded upon the virtues of Jekyll, describing it as the perfect approach to my problem. GitHub Landing Pages is a platform tucked away inside GitHub, and it lured me in with the three words all freshers would like to hear in this economy - completely free hosting. GitHub Pages ships freely with automatic deployment, meaning no manual server deployment, no PowerShell commands and a built-in SSL certificate. Every time I commit my code to the repository, the code will be automatically updated on the website as well. 

Jekyll is the engine underneath it. It is a static site generator which takes your templates and content, written in plain Markdown files and converts them to HTML files for a browser to understand. Think of it as you might think of React.js or Vue without the JavaScript complexities, without the build pipelines and without any backend. 

In short, you write, Jekyll assembles, and GitHub Pages publishes. 

What we are witnessing as internet and computer technology grow is not just tooling improvements, but a more sophisticated layer of abstraction being created. Each generation of web development tools asks for less and less from the person sitting on the keyboard. We first abstracted away machine code, then assembly, then HTML files and now infrastructure, configuration and architectural decisions. The developer's job is increasingly about what to build rather than the plumbing which works underneath it. 

This is a profound shift. This means that the portfolio you have been putting off building, the blog website you have been meaning to start and the project page for a new site idea, none of it requires the suffering you imagined. The tools are ready, the pipeline is free. The only thing left to do is commit - in every sense of the word.
